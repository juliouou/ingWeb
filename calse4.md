## Introducción a HTML5 y Estructura Semántica

**HTML5 (HyperText Markup Language versión 5)** es el lenguaje esencial que se usa para **estructurar y dar significado al contenido** de las páginas web. Actúa como el esqueleto de cualquier sitio.

* **Mecanismo:** El navegador web lee el archivo HTML y lo interpreta mediante un proceso llamado parseo, construyendo el **Modelo de Objeto del Documento (DOM)**. Este proceso dicta dónde se ubican el texto, imágenes, videos y enlaces.
* **Roles:** HTML se enfoca en la **estructura** y el **contenido**. El diseño visual se maneja con **CSS**, y la interactividad con **JavaScript**.

### 2. Especificación de Secciones (Etiquetas Semánticas)

HTML5 introdujo etiquetas **semánticas** (con significado claro) que permiten a los desarrolladores y a los motores de búsqueda entender mejor la estructura y la jerarquía del contenido de la página.

| Etiqueta Semántica | Función | Contenido Típico |
| :--- | :--- | :--- |
| **`<header>`** | Contenido introductorio de una sección o de la página completa. | Logotipo, título del sitio y búsqueda. |
| **`<nav>`** | Contiene los **enlaces de navegación** principales. | Menús de categorías y enlaces principales. |
| **`<main>`** | El **contenido único y central** del documento. (Se usa solo una vez por página). | La funcionalidad principal de la página (ej. el listado de libros de MEBOOK). |
| **`<section>`** | Agrupa contenido que es **temáticamente relacionado**. | Un conjunto de libros destacados, o una sección de "Preguntas Frecuentes". |
| **`<article>`** | Contenido **independiente y autosuficiente** (que podría ser distribuido solo). | Un *post* de blog, una noticia, o la ficha completa de un libro. |
| **`<aside>`** | Contenido **indirectamente relacionado** con el contenido principal (barra lateral). | *Widgets*, anuncios o enlaces relacionados. |
| **`<footer>`** | Información final de la sección o del documento. | Derechos de autor, política de privacidad, datos de contacto. |
